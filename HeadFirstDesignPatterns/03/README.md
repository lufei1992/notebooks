### 开放-关闭原则
>
- 类应该对扩展开放，对修改关闭。
>
- 我们的目标是允许类容易扩展，在不修改现有代码的情况下，就可搭配新的行为。如果实现这样的目标，有什么好处？这样的设计具有弹性可以应对改变，可以接受新的功能来应对改变的需求。
